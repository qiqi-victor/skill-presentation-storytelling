# 🎯 Presentation Storytelling Skill

> 面向演讲者的 AI 叙事助手 — 把素材变成有说服力的演讲

**版本**: v3.6 | **维护者**: 🐼🦐子 | **适用场景**: A 改造 / B 创作 / C 美化 / D 翻新 / E 局部修改

---

## 是什么

Presentation Storytelling Skill 是 OpenClaw Agent 的叙事设计专家模块，专注于**把已有材料变成演讲者自己的叙事**。

### 一句话

> 30 分钟演讲，本来要准备 8-16 小时；用这个 skill，2-3 小时搞定。

### 适用场景

| 情境 | 输入 | 核心价值 |
|------|------|---------|
| **A 改造型** | 散落素材 → 完整 PPT | 从零构建叙事 |
| **B 创作型** | 零想法 → 完整 PPT | 从空白到完整叙事 |
| **C 美化型** | 有稿子/略乱 → 精致 PPT | 叙事不变，优化表达 |
| **D 翻新型** | 别人的演讲 → 我的演讲 | 重构叙事，保留核心换表达 |
| **E 局部修改** | 只改几页 | 定向重做，不破坏整体结构 |

### 核心定位

PPT 专项三角色分工：
```
策划（本 Skill）→ 编剧（口语化）→ 美工（PPT制作）
```

策划负责理解材料、构建叙事线、设计节奏；编剧负责口语化表达；美工负责视觉实现。

## 使用方法

在 OpenClaw 对话中，直接描述你的需求：

```
帮我把这份 PPT 变成我自己的演讲：
[粘贴 PPT 内容或上传文件]

场合：XX 大会演讲
听众：XX 行业专业人士
目的：阐述 XX 观点，推动 XX 决策
时长：30 分钟
```

Skill 会自动识别场景类型，引导你完成：

1. **材料理解** — AI 消化 73 页 PPT，识别叙事线和重构机会
2. **叙事大纲** — 构建新的叙事结构（SCQA/STAR/金字塔等框架）
3. **故事脚本** — 逐页输出演讲主旨，而非逐字稿
4. **交接给编剧** — 口语化表达，交美工执行 PPT

---

## 流程控制

| 输入类型 | 动作 | 说明 |
|---------|------|------|
| 用户已有叙事大纲 | **Skip** Step 1-5 | 直接跳到故事脚本 |
| 用户已有故事脚本 | **Skip** 全部 | 直接输出评分 |
| 用户说"简版" | **Reduce** L2+L3 | 只用 Layer 1 普适方法论 |
| 用户说"行业版" | **Reduce** L3 | 加 Layer 2，跳过风格适配 |
| 用户指定页数 | **Adapt** | 按页数控制叙事节奏 |
